3 Windows and Content Pages > 3.2 Pop-Up Windows

3.2 Pop-Up Windows

Pop-up windows contain only secondary content pages. Peripheral tasks, such as composing a new message or managing an address book, are done in new pop-up windows to keep the main window stable and available for use (never swapped out).

Required

  • R1 Use a new pop-up window:
    • To focus on a specific task subsequent to the purpose of the main browser window.
    • When the main browser window is a "home" or "touchstone" page, a base where the user stays most of the time. The page may be reloaded to reflect updated data, but is never replaced with a different page.

    Example: In an e-mail application, the window that displays the Inbox and other mailboxes is the "touchstone" window for that application. Peripheral tasks, such as composing a new message or managing an address book, are done in new pop-up windows to keep the main window stable and available for use (never swapped out).

  • R2 Pop-up windows should contain secondary content pages only.

  • R3 To indicate the supplementary purpose of the content, pop-up windows should be smaller than parent windows.

  • R4 Pop-up windows should be large enough so that scrollbars are not displayed when the window is opened in the standard font size.

  • R5 To make pop-up windows chromeless (see definition on the previous page), suppress the menu bar and toolbars.

    When possible, suppress the status bar unless the page takes more than two seconds to load over a LAN (a gauge on the status bar provides feedback that an action is in process).

  • R6 Pop-up windows should open in the "golden mean" position relative to the entire screen (not the parent window). Use these formulas to calculate the vertical and horizontal coordinates to specify top and left location values in the JavaScript code which launches the pop-up window:

    Vertical = (screen.height - (screen.height/1.618)) - (pop-up_height/2)

    Horizontal = (screen.width - pop-up_width)/2

A pop-up window centered on the golden mean is more visually pleasing than the same window centered on the display's exact center.

If necessary move the pop-up window from the golden mean to ensure that it is positioned to be entirely contained within the visible screen area.

Note: Due to implementation limitations there is no provision for the parent window in these guidelines.

  • R7 Give pop-up windows the pop-up window masthead, as shown in Chapter 4, Mastheads. Since a pop-up window always contains secondary content pages, its masthead supports a secondary purpose. It does not contain any links or buttons.

  • R8 If a wizard has left-pane help or a list of steps, it must be shown in a pop-up window, not in the main browser window.

  • R9 When a user performs a task that changes the content of a minimized window, restore that window and bring it to the front of the display.

    • Example: When a pop-up window containing a dialog page is minimized, the user takes an action that displays new content in the dialog page. The implementation should force the minimized window to restore to its original size -- in front of other windows in the display. Without attention to proper coding, the windows might not behave in those ways.

  • R10 To prevent a window from being hijacked by a browser that is loading unrelated pages, name all windows (main and pop-up). The window name is not to be confused with its title; the name is invisible to the user and determined by the target attribute of a hyperlink (a href) tag or by the second argument of a Javascript window.open() method call.

  • R11 All pop-up windows should be coded so they will open even when the user has pop-up blocking enabled on their browser. They should also work without requiring the user to figure out that the application is trying to open pop-up windows which are being blocked, and adding the site to an exception list.

  • R12 To prevent pop-up blocking issues, never open windows in the onload of a document (i.e., the onload attribute of the body tag). In most cases the best solution is to open the pop-up window when the user clicks a button, link, or jump menu item. In effect, opening the pop-up in the onload of a document could make the window look like an advertisement to the browser (which will then be blocked).

Advised

  • A1 Determine use cases for opening pop-up windows, and follow them through the product. Pop-up windows could open for certain types of content and not others. It is more important to be consistent across pages of the same type, than across all types within an application.

  • A2 When any part of a window's chrome is suppressed, replace any needed browser-supplied functions with guideline-compliant controls in the window's content area.

    • Example: If the window's menu bar and toolbar are suppressed, users cannot print the window's contents by using the Print menu or by clicking on the Print button. If printing the contents of a window is deemed useful, print controls should be provided in the content area.

  • A3 Do not reuse a pop-up window to display a different content page. Reuse the window only to display a new instance of the same content page. A "different" page could be the same dialog content page for a different target. For example:

    Acceptable reuse: Clicking a Sort button followed by a Sort secondary content page opens into a pop-up window. The Sort page has an Apply button; the sorting criteria are applied to the main window and the Sort window remains open. If the user clicks the same Sort button in the main window again, the same Sort window should surface instead of another Sort window.

    However, do not reuse a popup window when two Sort windows are open if they were spawned by different Sort buttons. If the Sort buttons are above different tables in the main window, the button targets and the targets of the resulting Sort dialog content pages would differ.

    Discouraged reuse: When the user clicks a New button in the main window and the Sort window's contents is replaced with the New dialog page.

    • Note: Online help reuses the same window for all help topics and for the Glossary window.

  • A4 Do not open a pop-up window from another pop-up window. Instead, do one of the following:
    • Make the initial secondary content page appear in the main browser window. The next secondary content page would be the only one opened.
    • Redesign so that the additional secondary content page is unnecessary.
    • Use a wizard to contain all the steps.
