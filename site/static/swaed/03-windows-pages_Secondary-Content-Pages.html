3.8 Secondary-Content Pages

Secondary-content pages support interaction between a primary page or another secondary-content page. Context depends upon the parent page opened (could be a primary content page or a secondary-content page). They are usually dismissed by users quickly, after using for a specific purpose. There are several types of secondary-content pages. Major types include dialog and wizard pages as shown in the following table:

Required

  • R1 Secondary-content pages can appear in the main browser window, pop-up windows, or JavaScript windows.

  • R2 For buttons and breadcrumbs on secondary-content pages:
    • Provide a Cancel, Close, or "Back to Xxx" button to return the user to the originating page:

      Pop-up window: Provide a Cancel or Close button. A Close button is appropriate if there is no pending action to be cancelled.

      • Main browser window: Provide a Cancel or "Back to Xxx" button, where "Xxx" is the title (possibly abbreviated) of the originating page. A "Back to Xxx" button is appropriate when there is no pending action to be canceled, or if there is no data to save or lose.

    • Use breadcrumbs if the secondary-content page appears in the main browser window.
    • If the secondary-content page in the main browser window replaced other navigational elements (for example, tabs), then include them in the breadcrumb (for example, the active tab name). Provide additional contextual cues in the page if they would be useful.

  • R3 Titles should always be meaningful. They should provide consistent, direct, contextual information to ensure that users know which page, object, or action the page is supporting.

Advised

  • A1 Use buttons for secondary-content page actions (versus menus). Buttons are a way to distinguish secondary from primary content pages (where controls can include menus). Menus should not be necessary because secondary-content pages have a specific purpose.

  • A2 When a secondary-content page appears in a pop-up window, consider making windows single-use instead of multi-use:
    • Include action buttons such as OK that close the window after the action is invoked.
    • Avoid Apply buttons (or equivalents) that leave the window open.
    • Make the page multi-use (with Apply buttons or equivalents) if users would clearly benefit.

    • Note: This guideline differs from guidelines for the non-browser world. The non-browser world manages windows so that users can leave secondary windows open if they want to use them again. With Web browsers, pop-up windows can get lost or become orphaned. This guideline does not prohibit multi-use windows. It merely suggests that you judge whether users will value pop-up windows remaining open.

  • A3 Do not open a secondary-content page from another secondary-content page by swapping the parent secondary-content page. Instead do one of the following:
    • Open the additional secondary-content page into a pop-up window.
    • Redesign so that the additional secondary-content page is unnecessary.
    • Use a wizard to contain all the steps.

  • A4 Do not swap an entire window for a secondary-content page. This can be an effective way to grab the user's attention, but the context of the page that led to that secondary-content page can be lost. It is better to swap a portion of the window, such as the right pane. Another option is to open a pop-up window instead of swapping any portion of the original window.
