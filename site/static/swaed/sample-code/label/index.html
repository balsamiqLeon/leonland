Overview

This page includes sample HTML and styles for the label tag which are discussed in 7.3.1 Labels and 14.5 Validation Error Alerts. This tag uses the <label> HTML tag specified in HTML 4 to create an association between a form element and the text that is intended to be its label. Without the <label> tag, accessible technologies, such as screen readers, cannot determine when such an association exists. The situation is especially problematic when the intended label for the element is contained in a different table cell than the element itself, a situation which happens often with complex layouts, such as those used in our web applications.

Besides Section 508 compliance (see 1.4 Resources for Application Designers for more information), another benefit of using the label tag is that for IE and Netscape 6/7/Mozilla, clicking on text that has been associated with a form element via a label tag will activate the item (e.g., select a radio button) as if the form element itself had been clicked. This mimics traditional GUI behavior that has to date not made its way into web UI's.

The label HTML tag is used by wrapping <label for="IDofAssociatedFormElement"></label> around the text intended to be the label for the element. The ID for the associated element must be set and it must be specified in the "for" attribute. Typically, our labels will also require some context-specific text styling, so the appropriate style should be called in a <span> that wraps around the <label> as follows:

<span class="LblLev2Txt"><label for="AvailableNodes">Available Nodes:</label></span>
<select ID="AvailableNodes" size=15 multiple="true">
 <option selected>Node 1</option>
 <option>Node 2</option>
</select>

Form elements are often organized hierarchically into groups and subgroups. The labels on the individual elements as well as the groups are designed to visually support this hierarchy in that groups have bolder, heavier labels whereas individual elements within the groups have lighter labels. In the example below, Levels 1 and 2 would have a heavier appearance than Level 3. If form elements are not in a group or subgroup they have a default appearance of Level 2.
