Overview

This is the example HTML for properties pages and corresponds to Chapter 6, Content Area Structure and Layout. Many of the relevant details are spelled out in the chapter, so they won't be repeated here, but below is a summary of the general rules:

  • Jump links are added if the page "... is long enough to require significant scrolling under typical usage conditions"
    • If jump links are added, their name should be the same as the section name and should have a tooltip (set via title attribute) that says "Jump Down to the <section-name> Section"
    • If jump links are added, then "Back to Top" links should be added at the bottom of every section
  • "Indicates required field" text should only be shown if there are required fields on the page
    • Required field glyphs should be added to the right of the required element's label text
  • Fields with errors should be marked with a red error glyph to the left of the label and the text label should be red and bold
  • Buttons should be configurable and can range in number from 1 to 3 -- a likely default for properties pages would be "OK" and "Cancel"
  • Buttons should be repeated at the top and bottom of the page if the page is likely to require scrolling and only at the bottom if scrolling is not likely

See the release notes below for more implementation-specific information.

Release Notes

For section 508 compliance, the page sections are implemented using the fieldset and legend tags. These tags enable assistive technologies to associate groups of form elements as a related set. Unfortunately, Netscape 4 does not support this tag at all and Netscape 7 (as of July 2003) does not handle it particularly well (e.g., can't print & doesn't properly handle horizontal scrolling), so browser-specific solutions are required. For all browsers except IE, a <div class="ConFldSetDiv"> is used instead of the fieldset -- this sets the outer margins. Similarly, for all browsers except IE, a <div class="ConFldSetLgdSpn"> is used instead of the legend tag.

Another accessibility feature in these guidelines is the use of the label tag to associate text labels with their form elements. The various form element micro component tags already support this feature.

One side effect of the fieldset tag is that when it is used it creates a border around all of the items contained within. However, this behavior is overridden in the IE style sheet.
