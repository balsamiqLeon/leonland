Overview

This is sample HTML for the Full Page Alert component described in 14.3 Full Page Alerts. When used, this component comprises the main content area of a page, although it may appear in conjunction with navigational elements such as tabs and breadcrumbs. The sample HTML shown here assumes that the tag will use the page-title component for the page title portion of the design (i.e., no custom styles were created to show the page title for use with full page alerts).

There are four versions of the full page alert: Information, Error, Warning, and Question. In terms of appearance and HTML, the only difference between the four is the icon used in the page title and the page title text. All alerts have a header, message text and one or more buttons. Optionally, the alerts may also contain one or more form elements contained within to allow users to provide some additional required information (see the question example in the links below). The form elements contained in the alert are limited to the following:

  • A single label and Selectable List
  • A single label and Text Area, Text Field, or Password Field
  • A single label and Drop Down Menu
  • A single label and group of Radio Button
  • A single label and group of Checkboxes

The full page alert can also appear in a pop-up window. In this case, the window should be chromeless and should contain a pop-up window masthead. The initial window size should be developer configurable, but default to 640 by 300 px. The window should appear in the golden mean location.
