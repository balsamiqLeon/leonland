Overview

This is sample HTML and javascript for the horizontal and vertical versions of the add-remove idiom specified in 8.2 Add-and-Remove Idiom. The behavior specified in the javascript mostly conforms to the behavior set forth in the guidelines, but differs in a few key ways. See the release notes below for the details. The same HTML and javascript is used for all browsers, although there are some minor differences in the styles defined in the style sheets.

The Add and Remove idiom can appear in two orientations, horizontal and vertical. Within these orientations, there are other variations depending on the buttons to be included (see table below for required vs. optional buttons) and an option to include a meta-label (shown in the horizontal orientation image below). The buttons are grouped by function such that buttons with similar functions are close together and those with dissimilar functions are separated by additional white space. Basically, there are 3 possible groups: Add functions, Remove functions and Move functions.

Button Status Grouping
Add Required By itself or with Add All
Add All Optional, required if Remove All included With Add
Remove Required By itself or with Remove All
Remove All Optional With Remove
Move Up Optional, required if Move Down included With Move up
Move Down Optional, required if Move Up included With Move Down

Release Notes

The text labels on the list boxes also use the <label> tag to support accessibility. To use this feature the id for each of the list boxes must be set, and this id must be referenced by the label. See the label tag for more details on the use of this tag.
