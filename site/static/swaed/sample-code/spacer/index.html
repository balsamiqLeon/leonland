Overview

This is sample HTML for a very simple utility tag that simply creates a specified amount of vertical and horizontal "white space". The idea is that the tag will allow developers to specify three things:

  • Vertical Space - in pixels (defaults to 1 pixel)
  • Horizontal Space - in pixels (defaults to 1 pixel)
  • Whether the tag should force a line break and occupy its own line - (default is no)

This tag simply stretches a 1px x 1px transparent gif to the specified size. If the tag is specified to force a line break, the image is placed inside a <div> tag. Because the gif is transparent, the image will pick up the background color of whatever its parent container is (e.g., the page, a table cell, etc.). To ensure that screen readers ignore the image, its alt attribute must always be set as follows: alt=""

So if the developer specifies 20px of vertical and a line break, the tag would set down something like this:

<div><img src="../suntheme/images/other/dot.gif" width="1" height="20" alt=""></div>

If 10 pix of horizontal were specified, the tag would lay down something like this:

<img src="../suntheme/images/other/dot.gif" width="10" height="1" alt="">

Release Notes

Several implementation alternatives were evaluated in prior releases before deciding upon the spacer gif approach. These included the following:

  • Tables -- these create a lot of extra HTML, particularly for screen readers and will always force a line break
  • Empty div or span tags both with associated styles to handle the spacing increments (e.g., a 5px style and so on) and with inline style declarations (e.g., style="padding:5px 0px 0px 5px") -- but validators have a fit when they see empty tags
  • Use of div or span tags with contents such as non-breaking spaces -- even if the font for the non-breaking space is set to something very small, it was impossible to get much in the way of precision, it was dang near impossible to specify sizes under 5px, and required lots of extra style specification.

It proved impractical to remove them from the current release and so they remain.
